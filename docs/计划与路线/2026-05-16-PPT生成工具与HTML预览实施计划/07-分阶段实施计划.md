校验规则：

- `title` 非空，长度限制。
- `language` 只能是 `zh-CN` / `en-US`。
- `style` 首发只能是 `swiss`；`editorial` 只允许在二期候选 fixture 和 registry PoC 中出现，不能进入公开 `generate_ppt` schema。
- Swiss theme 只能是 `ikb` / `lemon` / `lemon_green` / `safety_orange` / `custom`；`theme:"custom"` 时 `custom_accent:"#RRGGBB"` 必填，其他 theme 不允许携带 `custom_accent`。
- Swiss layout 首发只允许白名单：`S01,S02,S03,S04,S08,S11,S15,S19,S21,S22`。
- 每页文本长度按 layout 限制，防止溢出。
- 图片槽位必须匹配 layout。
- 复刻 `validate-swiss-deck.mjs` 的关键约束到 spec 层：
  - 不允许未登记 layout。
  - `S15/S16/S22` 图片槽位必须声明比例。
  - 本地/内部图片必须绑定 `data-image-slot`。
  - SVG/diagram 文本必须进入结构化 label 字段，不允许 renderer 生成 SVG `<text>`。
  - 正文页标题默认左对齐，只有 statement/cover/closing 类 layout 可居中或巨字排版。
  - 7-8 页 deck 的 layout 多样性低于 6 个 S 编号时，M1 记录 warning，M2 首发闸门可升级为 error。
  - S08 地图变体首发拒绝，`map_variant:false`。

测试：

- `go test ./internal/presentation/spec -v`
- 覆盖非法 layout、缺标题、过长标题、错误图片槽位、额外字段、非法本地图片路径。
- 断言 JSON Schema 的 layout enum 与 validator registry 一致。
- 断言 JSON Schema 由 `layouts.json` + config 动态生成，不允许 `generate_ppt.go` 手写第二份 schema。
- 对同一组 valid/invalid fixtures 同时运行 JSON Schema 和 Go validator，断言失败 JSON pointer 一致。
- 断言 validation error 带 JSON pointer。
- 断言 `OutlineSpec` 到 `DeckSpec` 的接口类型不依赖具体 renderer/exporter。
- 断言每个公开 layout 的 `pptx_slots` 坐标在 13.333 × 7.5 画布内。
- 断言每个 text slot 都有 `overflow_policy`，且只使用 `reject/split_slide`。
- 断言中文标题字号分档按字符数生效，HTML/PPTX 共享同一组 `title_font_tiers`。
- 断言 Carbon spacing token 可通过 `spacingToInch` 转为 inch 坐标，且所有 public layout 坐标不漂移。
- 断言 8 页 fixture 的 layout 多样性 warning/error 行为符合阶段配置。
- 断言 `custom_accent` 只接受 `#RRGGBB`，拒绝 CSS 函数、颜色名、URL 和控制字符。

### Phase 2：HTML renderer

新增：

- `internal/presentation/renderhtml/renderer.go`
- `internal/presentation/renderhtml/swiss.go`
- `internal/presentation/renderhtml/templates/swiss.html`

实现策略：

- 不直接复制完整 `template-swiss.html` 的外部依赖。
- 提取其视觉 token、基础 CSS、版式骨架；首发不提取翻页 JS。
- 使用系统字体栈；如引用 `fonts.json` 中自托管字体，必须由后端内联或同源 asset 提供，不访问 Google Fonts。
- 图标只允许引用 `icons.json` inline path registry；不加载 Lucide runtime。
- 不输出 Motion One、MapLibre、Tailwind Play CDN 或任何外链 script。
- 所有 slide 通过 Go template 或 strings.Builder 受控生成。
- 输出 HTML 结构统一为：
  - `<meta name="hive-deck-version" content="1">`
  - `<section class="slide" data-slide-id="1" data-layout="S01">`
  - 可选 `<aside class="notes">...</aside>`
  - 不输出外链 script，不输出用户 raw HTML。
- 渲染后运行轻量 HTML 自检：
  - slide 数一致
  - 每页有 `data-layout`
  - 每页有 `data-slide-id`
  - head 有 `hive-deck-version=1`
  - 无外部 script
  - 无未转义用户 HTML

测试：

- `go test ./internal/presentation/renderhtml -v`
- fixture 渲染后包含正确 slide 数、主题变量、layout 标识。
- fixture 渲染后包含 `hive-deck-version=1`，并可被前端 `parseDeck` 同等规则解析。
- 使用 `golang.org/x/net/html` 做结构测试，断言无外链字体/script、无 `data-lucide`、无 Motion/MapLibre 依赖。

### Phase 3：PPTX exporter worker

新增：

- `tools/presentation-exporter/package.json`
- `tools/presentation-exporter/package-lock.json`

实现策略：

- 使用 PptxGenJS。
- `pptx.layout = 'LAYOUT_WIDE'` 或自定义 13.333 × 7.5 inch。
- 每个 Swiss layout 映射为 PPTX 原生对象：
  - 标题：`slide.addText`
  - 卡片/色块：`slide.addShape`
  - 图片：`slide.addImage`
  - 分割线：`slide.addShape(line)`
- 保留主题 token 到 PPTX 色值。
- 中文字体优先 `PingFang SC` / `Microsoft YaHei` fallback，英文 `Inter`/`Arial`。
- 实现 `spacingToInch(sp int) float64`，所有 layout 坐标优先来自 Carbon spacing token。
- 实现中文标题字号分档，按 `title_font_tiers` 选择 font size。
- 图标只允许从 `icons.json` 映射为 PptxGenJS shape/SVG data，禁止模型传入任意 SVG。
- 实现 `--health`：输出 `{ "ok": true, "protocol_version": 1, "pptxgenjs_version": "4.0.1" }`。
- worker stdout 只输出 JSON report，PPTX 写入 Go 指定 output path。

测试：

- `npm test` in `tools/presentation-exporter`
- 用 Node 测试生成 `.pptx`。
- 解压 PPTX 检查：
  - `[Content_Types].xml`
  - `ppt/presentation.xml`
  - `ppt/slides/slide1.xml`
  - 每页至少包含标题文本节点。
  - editable 模式不得只有 full-slide 图片。
- 对每个公开 layout 读取 `layouts.json` 的 `pptx_slots`，断言输出对象边界不越界。
- 对超长文本 fixture 断言 validator 拒绝，不能由 exporter 静默缩小字体。
- 中文 fixture 验证 PPTX 中字体声明包含中文 fallback，PowerPoint/Keynote/WPS 至少一种打开不乱码。
- notes fixture 验证 speaker notes 写入成功；若某环境不支持，必须输出 warning 且测试覆盖 warning。
- S08 map variant fixture 断言被 validator 拒绝或返回 `unsupported_layout_variant`。

### Phase 4：Go 侧 PPTX 封装与 asset 上传

实现策略：

- `Service.Generate(ctx, request) (PresentationResult, error)` 负责编排 validation、runtime、image resolve、HTML render、PPTX export、asset upload。
- `PresentationRunStore` 记录 run 状态、错误、耗时、asset URI 和统计字段。
- `Exporter.Export(ctx, request) (ExportResult, error)` 使用固定 worker 协议。
- 默认 worker path 从 config 读取，开发默认 `tools/presentation-exporter/src/exporter.mjs`。
- Go 控制超时、stdout/stderr、output path、最大输出。
- 图片 resolve 在 `resolving_images` 阶段集中完成，HTML/PPTX 共用 `resolved_assets`。
- 上传 DeckSpec JSON、HTML、PPTX 到 `asset.AssetService`，namespace 使用 `presentations/user/<owner_id>/run/<run_id>`，三者共享 `presentation_run_id` tag，但重试和 GC 以 run 表 URI 为准。
- asset upload owner 来自 `RuntimeContext`，不是模型输入。
- 同步生成超过 `sync_timeout_seconds` 时返回 `status:"running"`，由前端轮询 run。

实现策略：

- 使用独立 `tools.RegisterPPTGen(sc.MCPHost, sc.AssetService, presentationService, logger)` 注册，不扩展 `RegisterBuiltinTools` 的 variadic optional 依赖。
- 使用独立 `tools.RegisterDiagramGen(sc.MCPHost, sc.AssetService, diagramService, logger)` 注册 diagram 工具。
- Bootstrap 在 asset service 初始化后创建 `PresentationRunStore/DiagramRunStore` 和对应 service，再注册工具。
- Bootstrap 顺序固定为 `pgPool -> AssetService -> PresentationRunStore/DiagramRunStore -> PresentationService/DiagramService -> initAssetAccessResolver source_kind 分支 -> RegisterPPTGen/RegisterDiagramGen`。
- 注册 `GET /api/v1/presentation/runs/{run_id}`，用于前端轮询。
- `tool_choice_detector` 或等价 host intent guard 增加 artifact 意图识别。用户明确请求 PPT/演示文稿/slide deck/下载 PPTX 时，不能返回 `auto`；用户明确请求 Mermaid/脑图/流程图/架构图/可下载 SVG/PNG 时，不能返回 `auto`。用户明确只要 Mermaid 源码且不需要预览/下载时，可不强制工具调用。
- 工具 input schema 引用 `DeckSpec` 强 schema。
- 工具 output schema 声明 `kind/run_id/status/title/mode/deck_spec_asset_uri/html_asset_uri/pptx_asset_uri/slide_count/warnings/validation`，但强校验必须由 Go typed result marshal 和单测保证，不能依赖 Host output schema。
- 工具支持 `status:"running"` 输出；前端可用 run id 查询状态。
- 工具错误分三类：
  - invalid_spec：模型可修复
  - exporter_unavailable：部署/配置问题
  - asset_unavailable：服务未初始化
- asset resolver 在现有 `internal/bootstrap/asset.go` 的 `assetAccessResolver.CanResolveAsset` 内增加 `generated_presentation` / `generated_diagram` 分支，按 purpose/format/session 校验；当前 `ResolveAsset` 不是 resolver chain，不能新增一个并行 resolver。
- 修改现有 `/api/v1/assets/proxy` handler 的 `Content-Disposition` 逻辑，支持签名覆盖的 `purpose/disposition/filename`；不要新增下载 endpoint。
- `presentation_download` / `diagram_download` 首发所有 provider 都走 Hive signed proxy，避免 S3/MinIO signed URL 无法设置 attachment header。
- Config 增加 presentation/diagram 配置，校验 `sync_timeout_seconds < timeout_seconds` 且小于 `RuntimePolicy.ToolTimeout`。
- Dockerfile runtime stage 显式复制并安装 `tools/presentation-exporter` 和 `tools/diagram-exporter`。

测试：

- 工具注册测试。
- `tool_choice_detector` 测试覆盖“帮我做一份 PPT”“生成一个可下载的 slide deck”“画一个 Mermaid 流程图并下载 SVG”“生成一个脑图”，上述请求不能返回 `auto`。
- `tool_choice_detector` 测试覆盖“给我 Mermaid 源码即可，不需要预览/下载”可以不强制工具。
- invalid spec 返回 `IsError=true` 且包含结构化错误。
- fake service 返回 asset URI 后，tool result JSON 满足 typed parser，且体积小于 5KB，不含完整 HTML。
- fake service 返回 running 后，tool result JSON 满足 pending parser。
- run 查询 API owner 通过、错 owner 403、错 session 403、无 session 同 owner 通过。
- `presentation_download` 下载 pptx 通过，下载 html 拒绝。
- `presentation_preview` 预览 html 通过，错 session 拒绝。
- 普通 `user_asset` purpose 不能 resolve presentation asset。
- `presentation_download` proxy URL 篡改 `purpose/filename/disposition` 后 403。
- 现有普通 asset proxy 仍保持 inline，不因 presentation/diagram 修改回归。
- 过期 run 的 GC dry-run 能列出对应三类 asset，未过期 run 不被列出。
- 同一 session 连续 3 次 model-recoverable 错误后，工具返回 `recoverable=false/recoverable_by=user`。

  - 支持键盘翻页和全屏演示。

### Phase 8：端到端薄片与首发验收

验收用例：

1. 用户请求“做一份 6 页瑞士风 PPT”。
2. 模型调用 `generate_ppt`。
3. 工具生成 HTML 和 PPTX asset。
4. Chat 工具结果卡展示“预览 / 下载 PPTX”。
5. 点击预览打开右侧 Canvas，可翻页。
6. 点击下载得到 `.pptx`。
7. 后端用 `fileconv.extractPPTX` 或 zip XML 验证每页标题文本存在。
8. 错 session 下载 PPTX 返回 403。
9. worker 关闭时工具返回 `exporter_unavailable`，Chat 不崩溃。
10. running -> failed 时 Chat 显示结构化错误、run id 和 disabled 下载按钮。
11. 页面刷新后，历史 presentation artifact 通过 run id 恢复预览/下载状态。
12. HTML 生成成功但 PPTX export 失败时，Canvas 可预览 HTML，PPTX 下载 disabled。
13. 移动端打开 DeckRenderer，缩略图条/notes/toolbar 不重叠。
14. 篡改 proxy URL 的 `purpose/filename/disposition` 后返回 403。
15. worker crash、stdout 非 JSON、stdin 写入失败均返回结构化错误且记录日志。
16. 用户请求“画一个 Mermaid 流程图”，模型调用 `generate_diagram`。
17. 工具生成 Mermaid source 和 SVG asset，Chat 工具结果卡展示“预览 / 下载 SVG / 下载源文件”。
18. 点击预览打开右侧 Canvas，SVG 非空、可缩放、源码 tab 可读。
19. 点击下载得到 `.mmd` 和 `.svg`。
20. Mermaid 恶意 SVG fixture 不执行脚本，sanitize 后下载文件不含危险节点/属性。
21. 用户请求“生成脑图”时，如果 Mindmap 已启用，工具生成 MindMapSpec 并提供 Canvas 预览和 `.mm.json/.svg` 下载；如果未启用，返回 `unsupported_diagram_type` 或 `diagram_exporter_unavailable`，Chat 不崩溃。
