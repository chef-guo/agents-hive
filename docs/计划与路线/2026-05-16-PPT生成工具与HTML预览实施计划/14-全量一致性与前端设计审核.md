## 3. 一致性结论

当前执行口径已经收敛为：

- M1/M2 主线是 **PPT 可编辑生成 + Mermaid Canvas/source/SVG 闭环**。
- Mindmap 不是可选想法，但默认是 **M3 用户可见能力**；M0 必须冻结 `MindMapSpec`、fixtures、安全限制和 tool intent 规则。
- PPTX 默认下载必须是 PptxGenJS 原生对象生成的 editable PPTX；visual PPTX 只能作为后续显式模式，不能伪装成可编辑。
- Artifact 工具结果必须走 `ToolAdapter -> GeneratedArtifactResultCard -> Canvas`，不能只改 standalone `ToolResultCard`。
- 下载语义必须走 signed Hive proxy，`purpose/disposition/filename` 参与 HMAC；前端只触发 URL，不依赖 `<a download>` 决定文件名或 attachment。
- M2 presentation 的普通用户下载入口只有 editable PPTX；HTML 是 preview asset，不用 `presentation_preview` 做 attachment 下载。
- Asset provenance 以 run 表为 canonical truth；run-scoped namespace 是首发硬约束，tags 只做排查线索。
- Swiss M2 首发是 10 个 layout：`S01,S02,S03,S04,S08,S11,S15,S19,S21,S22`，fixture 是 `swiss-basic-10.json`。
- `theme:"custom"` + `custom_accent:"#RRGGBB"` 已进入 schema 和校验口径。

前端发布阻断项已写入 `05` 和 `07`：

- 没有 desktop/tablet/mobile Playwright 截图或等价视觉验收，阻断。
- running/failed/partial/success 任一状态缺测试，阻断。
- 移动端 toolbar、notes、缩略图、下载菜单重叠，阻断。
- icon-only 按钮缺 `aria-label`、tablist/focus/键盘导航缺失，阻断。
- 新 UI 绕过 `ToolAdapter` 或把所有 artifact 塞进 `HtmlRenderer`，阻断。

## 5. 仍需实施前二次确认

这些不是当前文档冲突，但进入 Phase 0 前仍需要具体化：

1. 是否把 Mindmap 从 M3 前移到 M2。如果前移，不能只开前端入口，必须前移 worker、asset purpose、history restore、安全负测和 Playwright。
2. Phase 6 前是否生成视觉 mockup/线框对比板。当前计划已经有设计矩阵，但真实布局仍应在实现前用截图或 mockup 做一次产品确认。
3. `DESIGN.md` 当前字体记录里仍提到 Google Fonts/CDN；PPT/Diagram 生成链路禁止外链字体，前端应用自身字体策略不在本计划中重构，但实现时不能把外链依赖带进 worker/iframe artifact。
4. `docs/` 当前被 `.gitignore` 忽略，Phase 0 前必须让本计划和后续架构/运维文档进入可追踪路径。

## 6. 退回条件

实现评审中如果出现以下任一情况，直接退回计划修正：

- 重新出现 Swiss 8 作为 M2 首发质量目标。
- `generate_ppt` 或 `generate_diagram` 返回完整 HTML/SVG/source 而不是轻量 manifest。
- 使用全局 `presentations` / `diagrams` namespace。
- 只改 `ToolResultCard`，没有接入 `ToolAdapter`。
- 下载 PPTX/SVG/source 依赖前端 blob 或直签 URL，绕过 signed Hive proxy。
- 把 Mindmap 半成品入口暴露给普通用户，但没有 M2 前移的完整后端、前端、安全和 E2E 验收。
- DeckRenderer/DiagramRenderer 缺移动端、键盘、ARIA 或错误状态验收。
